# ШТОЛЛИ — сайт мастерской скульптуры и художественного литья

Продакшен-сайт мастерской **ШТОЛЛИ** (Сочи): скульптура, художественное литьё из
бронзы, латуни, бетона и гипса. Одностраничный статический сайт с премиальным
редакторским моушен-дизайном.

> Пересборка дизайн-макета `Штолли.dc.html` (Claude Design) в полноценный
> production-ready сайт: реальные изображения, продвинутая анимация, доступность,
> перформанс.

---

## TL;DR — как открыть

Это **статический сайт без сборки**. Любой из вариантов:

```bash
# Вариант 1 — локальный сервер (рекомендуется: корректная загрузка шрифтов/CDN)
python -m http.server 8131
# открыть http://127.0.0.1:8131

# Вариант 2 — просто двойной клик
# открыть index.html в браузере (нужен интернет для GSAP/Lenis/Google Fonts)
```

Деплой: загрузить папку целиком на любой статический хостинг (Netlify, Vercel,
GitHub Pages, обычный nginx/Apache). Подробности — [docs/DEPLOYMENT.md](docs/DEPLOYMENT.md).

## Стек

| Слой | Технология |
|---|---|
| Разметка | Семантический HTML5, без фреймворка |
| Стили | Чистый CSS, дизайн-токены через custom properties, 3 модуля |
| Скрипты | Vanilla JS (classic scripts, без бандлера — работает и по `file://`) |
| Плавный скролл | [Lenis](https://lenis.darkroom.engineering/) 1.3.23 (CDN) |
| Анимация | [GSAP](https://gsap.com/) 3.15 + ScrollTrigger + SplitText (CDN) |
| Шрифты | Playfair Display + Manrope (Google Fonts) |

Почему статика и CDN — см. раздел «Решения и trade-offs» ниже и
[docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).

## Возможности

- **Кинематографичный hero** — построчное раскрытие заголовка (SplitText + mask),
  clip-path reveal изображений, многослойный параллакс.
- **Плавный скролл** (Lenis), синхронизированный с GSAP ScrollTrigger.
- **Scroll-driven reveal** всех секций пачками (ScrollTrigger.batch) со стаггером.
- **Бегущая строка** материалов (бесшовный marquee), пауза вне экрана.
- **Анимированные счётчики** статистики.
- **Magnetic-кнопки** и **CTA** (следование за курсором).
- **Список проектов** с плавающим превью у курсора.
- **Галерея-масонри** (22 работы) с lightbox: клавиатура, стрелки, ESC, focus-trap.
- **Прелоадер «розлив бронзы»**.
- **Адаптив** 360 → 1920, мобильное меню с focus-trap и `inert`.
- **Доступность**: семантика, ARIA-лендмарки, `prefers-reduced-motion`, прогрессивное
  улучшение (контент виден без JS / при сбое CDN).

## Решения и trade-offs

| Решение | Обоснование | Обратимость |
|---|---|---|
| Статика без сборки | Маркетинговый одностраничник; нулевой порог входа, тривиальный деплой, открывается по `file://` | легко обратить |
| GSAP/Lenis через CDN | Без npm/бандлера; кэш CDN; быстрый старт. Требует интернет для библиотек | легко обратить → self-host: [DEPLOYMENT.md](docs/DEPLOYMENT.md) |
| Классические `<script>` (не ESM) | ESM-импорты не работают по `file://` (CORS); classic + namespace `window.Shtolli` работают везде | дорого обратить |
| Галерея статична в HTML (не из JS) | SEO, работа без JS, корректные `alt` | легко обратить |
| Форма без бэкенда | Out of scope; собирает `mailto:` и показывает успех. Подключение Formspree/Telegram-бота — точечная правка | легко обратить → [CONTENT-GUIDE.md](docs/CONTENT-GUIDE.md) |
| Пересжатие изображений в webp | 13.3 МБ → 3.2 МБ (24%); см. [PERFORMANCE.md](docs/PERFORMANCE.md) | легко обратить (исходники в `img/`) |

## Админ-панель заявок (`admin.html`)

Единое окно для ведения заявок: канбан-воронка от первого обращения до
выдачи готовой работы. Открывается по адресу `/admin.html` (в навигацию сайта
не выведена, помечена `noindex`).

**Доступ.** Код по умолчанию — `shtolli2026`. Поменять: `ADMIN_PASS` в
[`assets/js/admin.js`](assets/js/admin.js). ⚠️ Это **демо-замок на стороне
браузера**, не настоящая авторизация (код виден в исходниках). Для реального
многопользовательского доступа нужен бэкенд — см. ниже.

**Откуда берутся заявки.** Квиз (`quiz.html`) и форма контактов (`index.html`)
при отправке автоматически сохраняют заявку в общий слой
[`assets/js/leads-store.js`](assets/js/leads-store.js) (`window.ShtolliLeads`,
хранилище — `localStorage`). Существующие каналы (WhatsApp / Telegram / почта)
продолжают работать как прежде — это дополнительный учётный канал.

**Этапы воронки** (`ShtolliLeads.STAGES`):

1. Новая заявка
2. Консультация
3. Согласование эскиза / 3D
4. Договор и предоплата
5. Литьё / производство
6. Финишная обработка
7. Готово / выдача
8. Архив (отказ / неактуально) — вне основной воронки

**Возможности.** Перетаскивание карточек между этапами (drag-and-drop) и
кнопки ◀ ▶; карточка → детали (контакты, параметры заказа, история этапов,
заметки); поиск, фильтр по источнику, архив, экспорт всех заявок в JSON.

**Ограничение демо-режима.** Данные живут в `localStorage` **одного браузера**.
Заявка, отправленная клиентом с его устройства, не попадёт в админку владельца —
для этого нужен общий бэкенд. Заложен явный шов: присвойте
`ShtolliLeads.remote = { send: function (lead) { /* fetch в Supabase / Telegram-бот */ } }`
— и каждая новая заявка будет уходить на сервер. Рекомендуемый апгрейд —
Supabase (работает с GitHub Pages, бесплатный тариф, реальная авторизация).

## Деплой (GitHub Pages)

Сайт статический — публикуется как есть. Источник Pages: ветка `main`, корень `/`.
После пуша Pages собирает сайт; служебные файлы (`.dc.html`, `image-slot.js`,
`support.js`) — артефакты исходного дизайн-макета, на работу сайта не влияют.
