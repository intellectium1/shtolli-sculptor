# Доступность (WCAG 2.2 AA)

Сайт построен по принципу прогрессивного улучшения и проверен по ключевым критериям
WCAG 2.2 уровня AA.

## Семантика и структура

- Лендмарки: `<header>`, `<nav aria-label>`, `<main id="main">`, `<section
  aria-labelledby>`, `<footer>`.
- Корректная иерархия заголовков: один `<h1>` (hero) → `<h2>` секций → `<h3>` карточек.
- Каждая секция связана с заголовком через `aria-labelledby`.
- Skip-link «Перейти к содержимому» (`.skip-link`) — первый фокусируемый элемент,
  виден при фокусе, ведёт на `#main`.
- `<html lang="ru">`.

## Клавиатура и фокус

- **Все интерактивные элементы** достижимы с клавиатуры; кликабельные изображения
  галереи — настоящие `<button>` (не `div` с обработчиком).
- **`:focus-visible`** — заметная бронзовая обводка с отступом (видна только при
  навигации с клавиатуры, не мешает мыши).
- **Мобильное меню**: focus-trap (Tab/Shift+Tab внутри), Escape закрывает, фокус
  возвращается на кнопку-бургер; фон (`<main>`) получает `inert`; `aria-expanded`
  и `aria-label` кнопки меняются по состоянию.
- **Lightbox**: focus-trap, Escape закрывает, фокус возвращается на открывшую работу;
  стрелки ←/→ листают; фон помечается `inert`; `role="dialog"`, `aria-modal="true"`.

## Движение и `prefers-reduced-motion`

- Класс `html.reduce-motion` ставится **до отрисовки** (inline-скрипт в `<head>`),
  поэтому при системной настройке «Reduce motion» анимации не запускаются вообще
  (ни Lenis, ни GSAP), а контент сразу виден.
- Дополнительный CSS-`@media (prefers-reduced-motion: reduce)` обнуляет длительности
  переходов/анимаций как защитная сетка.
- Счётчики в этом режиме сразу показывают финальные значения; бегущая строка статична.

## Формы

- У каждого поля — связанный `<label for>`.
- Валидация: при ошибке поле получает `aria-invalid="true"`, текст ошибки выводится
  рядом; визуально — класс `has-error`.
- Статус отправки — в контейнере с `aria-live="polite"` (скринридер озвучит).

## Изображения

- Все содержательные `<img>` имеют осмысленный `alt` с названием работы и материалом
  (напр. «Скульптура Николая Чудотворца — бронза, патинирование, 1 м»).
- Чисто декоративные элементы (кольца CTA, разделители) помечены `aria-hidden="true"`
  или реализованы как фон.

## Цветовой контраст

Пересчитан и приведён к AA (нормальный текст ≥ 4.5:1, крупный ≥ 3:1):

| Текст | Цвет / фон | Контраст | Статус |
|---|---|---|---|
| Body | `--c-muted #5c554a` / cream | ~6.6:1 | ✅ AA |
| Мета, подписи | `--c-faint #6e675b` / cream | ~5.0:1 | ✅ AA *(затемнён с #837b6e = 3.75:1)* |
| Кикеры, eyebrow, «Подробнее», активная навигация | `--c-bronze-ink #855d33` / cream | ~5.2:1 | ✅ AA *(отделён от декоративной бронзы)* |
| Текст на тёмном | `--c-on-dark-faint #9a9183` / ink | ~5.4:1 | ✅ AA |
| Акцент на тёмном | `--c-bronze-light #cdb494` / ink | ~8.4:1 | ✅ AAA |
| Заголовки | `--c-ink #211d17` / cream | ~13:1 | ✅ AAA |

Декоративная бронза `--c-bronze #93673a` (≈4.45:1 на кремовом) теперь используется
только для **крупного** текста (акцент «литьё», hover крупных serif-заголовков —
проходят как large text ≥3:1), фоновых заливок кнопок (белый текст на бронзе) и
графики (линии, точки), но **не** для мелкого основного текста.

> **Известное декоративное исключение:** крупные бледные порядковые числа карточек и
> строк (`01/02/03`, цвет `--c-bronze-light` на кремовом, ~1.8:1) трактуются как
> декоративная нумерация — тот же порядок передан DOM-структурой и заголовками. При
> необходимости строгого соответствия их можно затемнить до `--c-bronze-ink`.

## Прогрессивное улучшение (отказоустойчивость)

- Без JS — весь контент виден, галерея и навигация работают (нативный скролл +
  `scroll-margin-top` под фикс-навбар), бегущая строка анимируется CSS-ом.
- Если CDN-библиотеки не загрузились — `html.motion-off` возвращает все reveal-элементы
  в видимое состояние; сайт остаётся полностью функциональным.

## Тач/указатель

- Magnetic-эффекты и плавающее превью включаются только при точном указателе
  (`hover: fine` / `maxTouchPoints === 0`) — на тач-устройствах не мешают.
- Целевые размеры интерактивных элементов: кнопки ≥ 44–46px по высоте; бургер 44×44.

## Чек-лист ручной проверки

- [ ] Пройти всю страницу только Tab — фокус виден, порядок логичен, ловушек нет.
- [ ] Открыть меню и lightbox с клавиатуры, закрыть Escape — фокус вернулся на триггер.
- [ ] Включить «Reduce motion» — анимаций нет, контент читается, счётчики финальные.
- [ ] Отключить JS — контент и навигация работают.
- [ ] Проверить контраст инструментом (axe DevTools / Lighthouse a11y).
- [ ] Скринридер: озвучивание лендмарков, заголовков, `alt`, статуса формы.
