## Оптимизация изображений

Исходники (`img/`) пересжаты скриптом [`optimize_imgs.py`](../optimize_imgs.py) в
`assets/img/*.webp`:

- коррекция EXIF-поворота (`ImageOps.exif_transpose`);
- ресайз по большей стороне (hero ≤1700px, галерея ≤1400px, превью ≤1200px);
- экспорт WebP, quality 82, method 6.

**Результат: 13 357 КБ → 3 233 КБ (24% от исходного), 26 файлов.**

Самые тяжёлые исходники ужались радикально:

| Файл | Было | Стало |
|---|---|---|
| Наградные знаки | 2489 КБ | 93 КБ |
| Перо (Бестужев) | 1631 КБ | 131 КБ |
| Халк | 941 КБ | 80 КБ |
| Подставка для вина | 901 КБ | 184 КБ |

Финальные размеры — в [`assets/img/_manifest.json`](../assets/img/_manifest.json).

## LCP (главное изображение)

- Hero-картинка `hero-main.webp` предзагружается:
  `<link rel="preload" as="image" href="…/hero-main.webp" fetchpriority="high">`.
- На самом `<img>` — `fetchpriority="high"`, `loading` не `lazy` (above the fold).
- **Прелоадер не задерживает LCP**: картинка грузится за оверлеем, а оверлей снимается
  быстрым fade. В reduced-motion оверлей убирается мгновенно.

## CLS (сдвиги макета)

- У **всех** `<img>` заданы `width`/`height` → браузер резервирует место.
- Рамки галереи и hero используют `aspect-ratio` (`--ar`, `4/5`, `3/4`, `1/1`) →
  коробка известна до загрузки картинки; фон `--img-bg` исключает «вспышку».
- Шрифты — `display=swap` + `preconnect` к `fonts.gstatic.com`; `ScrollTrigger.refresh()`
  после `document.fonts.ready` пересчитывает позиции после подмены шрифта.

## INP / TBT (отзывчивость)

- Один общий rAF-цикл (Lenis через `gsap.ticker`), без конкурирующих циклов.
- Скролл-слушатель навбара — `{ passive: true }`.
- `ScrollTrigger.batch` + `once:true` — нет повторных колбэков на каждый кадр скролла.
- Анимируются только compositor-свойства (`transform`/`opacity`/`clip-path`) — работа
  уходит с main-thread на GPU-композитор.
- `will-change` включается перед анимацией и снимается (`clearProps`) после — не держим
  лишние слои в видеопамяти.

## Загрузка ресурсов

- CSS — 3 небольших файла (render-blocking, но крошечные после gzip; FOUC исключён).
- CDN-скрипты (GSAP×3 + Lenis) — `defer` (не блокируют парсинг/отрисовку).
- `preconnect` к Google Fonts.
- Картинки below-the-fold — `loading="lazy"` + `decoding="async"`.

## Идеи для дальнейшего ускорения

1. **`srcset`/responsive images**: сгенерировать 2-3 ширины на картинку и отдавать по
   размеру вьюпорта (сейчас одна оптимизированная ширина на слот).
2. **Blur-up плейсхолдеры**: крошечный base64-превью под каждую картинку для мягкого
   проявления (сейчас — сплошной `--img-bg`).
3. **Self-host шрифтов и библиотек** с `preload` критичного начертания — убирает
   сторонние коннекты ([DEPLOYMENT.md](DEPLOYMENT.md)).
4. **Lightbox**: подгружать оригинал по требованию (сейчас переиспользуется
   оптимизированная webp ~1400px — этого достаточно для 80vh).

## Как проверять

```bash
# Lighthouse (Chrome DevTools → Lighthouse, или CLI)
npx lighthouse http://127.0.0.1:8131 --view --preset=desktop
npx lighthouse http://127.0.0.1:8131 --view   # mobile
```

Проверяйте на «холодном» кэше; смотрите LCP-элемент (должен быть hero-картинка) и
отсутствие неожиданных сдвигов (CLS).
